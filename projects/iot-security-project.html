IoT Security Assessment & Penetration Testing

Comprehensive security evaluation of IoT devices using Kali Linux and network analysis tools

Spring 2025 Security Assessment Network Analysis

Security Assessment Overview

Security Challenge

IoT devices often have weak security configurations and default credentials, making them vulnerable to attacks. This project aimed to identify and exploit these vulnerabilities in a controlled environment.

Assessment Goal

To perform comprehensive security testing on IoT devices, identify vulnerabilities, and develop detection mechanisms for common IoT-based attacks.

Test Environment

Kali Linux penetration testing platform with simulated IoT devices (smart cameras, sensors) and a controlled network environment.

Security Assessment Methodology

Phase 1: Reconnaissance

  • Network scanning with Nmap to identify connected devices
  • Service enumeration and port scanning
  • OS fingerprinting and device identification
  • Vulnerability database matching

Phase 2: Vulnerability Analysis

  • Default credential testing using Hydra
  • Firmware analysis and reverse engineering
  • Communication protocol analysis (MQTT, CoAP)
  • Encryption strength testing with OpenSSL

Phase 3: Exploitation

  • SYN Flood DoS attack simulation
  • Man-in-the-Middle (MITM) attacks
  • Credential harvesting and session hijacking
  • Privilege escalation attempts

Phase 4: Detection & Mitigation

  • IDS rule development for IoT traffic patterns
  • Anomaly detection using machine learning
  • Security hardening recommendations
  • Incident response procedures

Network Architecture

# Network Scanning Script
import nmap
import socket
from scapy.all import *

class IoTScanner:
    def __init__(self, target_range):
        self.target_range = target_range
        self.nm = nmap.PortScanner()
    
    def discover_iot_devices(self):
        """Scan network for IoT devices"""
        print(f"Scanning network: {self.target_range}")
        
        # Perform comprehensive scan
        scan_results = self.nm.scan(
            hosts=self.target_range,
            arguments='-sS -sV -O -T4 --script=vuln'
        )
        
        iot_devices = []
        for host in self.nm.all_hosts():
            if self.is_iot_device(host):
                device_info = {
                    'ip': host,
                    'hostname': self.nm[host].hostname(),
                    'state': self.nm[host].state(),
                    'os': self.nm[host]['osmatch'][0]['name'] if 'osmatch' in self.nm[host] else 'Unknown',
                    'ports': self.get_open_ports(host)
                }
                iot_devices.append(device_info)
        
        return iot_devices
    
    def is_iot_device(self, host):
        """Identify IoT devices based on open ports and services"""
        common_iot_ports = [1883, 8883, 5683, 5684, 8884]  # MQTT, CoAP ports
        for port in common_iot_ports:
            if self.nm[host].has_tcp(port):
                return True
        return False
    
    def get_open_ports(self, host):
        """Extract open ports and services"""
        ports = []
        for proto in self.nm[host].all_protocols():
            lport = self.nm[host][proto].keys()
            for port in lport:
                service = self.nm[host][proto][port]['name']
                ports.append(f"{port}/{proto} ({service})")
        return ports

DoS Attack Detection

# SYN Flood Detection with Scapy
from scapy.all import *
import time
from collections import defaultdict

class DoSDetector:
    def __init__(self, threshold=100, window=10):
        self.threshold = threshold  # Max SYN packets per window
        self.window = window        # Time window in seconds
        self.syn_counts = defaultdict(int)
        self.last_reset = time.time()
    
    def analyze_packet(self, packet):
        """Analyze packets for SYN flood patterns"""
        current_time = time.time()
        
        # Reset counters if window expired
        if current_time - self.last_reset > self.window:
            self.syn_counts.clear()
            self.last_reset = current_time
        
        # Check for SYN packets without ACK (SYN flood indicator)
        if packet.haslayer(TCP):
            tcp_layer = packet.getlayer(TCP)
            if tcp_layer.flags == 'S':  # SYN flag set
                src_ip = packet[IP].src
                self.syn_counts[src_ip] += 1
                
                # Check if threshold exceeded
                if self.syn_counts[src_ip] > self.threshold:
                    print(f"[ALERT] Possible SYN Flood from {src_ip}")
                    print(f"  SYN packets in last {self.window}s: {self.syn_counts[src_ip]}")
                    self.trigger_mitigation(src_ip)
                    return True
        
        return False
    
    def trigger_mitigation(self, malicious_ip):
        """Implement mitigation measures"""
        # Block IP using iptables
        import subprocess
        subprocess.run([
            'iptables', '-A', 'INPUT', '-s', malicious_ip, '-j', 'DROP'
        ])
        print(f"  [MITIGATION] Blocked {malicious_ip} via iptables")
        
        # Send alert to security team
        self.send_alert(malicious_ip)

# IoT Device Behavioral Profile
class IoTBehaviorAnalyzer:
    def __init__(self):
        self.profiles = {}
        self.threshold = 2.0  # Standard deviation threshold
    
    def build_profile(self, device_id, network_data):
        """Build normal behavior profile"""
        profile = {
            'avg_packet_size': np.mean(network_data['packet_sizes']),
            'avg_packets_per_min': np.mean(network_data['packet_rates']),
            'common_destinations': Counter(network_data['destinations']).most_common(5),
            'protocol_distribution': Counter(network_data['protocols'])
        }
        self.profiles[device_id] = profile
        return profile
    
    def detect_anomalies(self, device_id, current_data):
        """Detect deviations from normal behavior"""
        if device_id not in self.profiles:
            return False
        
        profile = self.profiles[device_id]
        
        # Calculate deviation scores
        packet_size_dev = abs(current_data['avg_packet_size'] - profile['avg_packet_size'])
        packet_rate_dev = abs(current_data['packet_rate'] - profile['avg_packets_per_min'])
        
        # Normalize deviations
        normalized_dev = (packet_size_dev + packet_rate_dev) / 2
        
        return normalized_dev > self.threshold

Security Recommendations & Reflection

Key Vulnerabilities Identified

  1. Weak Authentication: Default credentials and lack of multi-factor authentication
  2. Unencrypted Communications: Plaintext transmission of sensitive data
  3. Insecure Network Services: Unnecessary open ports and vulnerable protocols
  4. Lack of Device Hardening: Minimal security configurations and outdated firmware

Security Hardening Recommendations

Implement strong authentication with certificate-based access
Enable encryption for all communications (TLS 1.3)
Segment IoT devices on separate VLANs
Regular firmware updates and vulnerability scanning
Implement network monitoring with IDS/IPS solutions

Lessons Learned

This project demonstrated how easily IoT devices can be compromised and the importance of security-by-design principles. The most significant takeaway was that many vulnerabilities stem from basic security hygiene failures rather than complex exploits.

Future Work: Develop an automated IoT security assessment tool that can scan home networks and provide actionable security recommendations to non-technical users.
